# DHIS2 EPI Prototype — Setup Guide

**Project:** UNICEF AHEAD / Gates Foundation AI Fellows  
**Scope:** Ethiopia immunization data quality sandbox  
**DHIS2 version:** 2.40 (docker image `dhis2/core:2.40`)  
**Date built:** May 2026

---

## Overview

This guide documents exactly how the DHIS2 prototype was built. It is intended to be reproducible end-to-end from a clean Docker environment.

The prototype covers:

- Ethiopia → Amhara → North Gondar Zone → 5 woredas → 12 facilities (5-level hierarchy)
- WHO EPI Aggregate Metadata Package v1.1.0 (routine vaccine delivery forms)
- Role-based users scoped to each administrative level
- 6 months of synthetic EPI data (Nov 2025 – Apr 2026) with 3 seeded data quality issues

---

## Prerequisites

- Docker Desktop installed and running
- Python 3.x (stdlib only — no pip installs required)
- ~4 GB disk space for DHIS2 + PostgreSQL volumes

## Phase 1 — Start DHIS2

### 1a. docker-compose.yml

```yaml
services:
  db:
    image: ghcr.io/baosystems/postgis:13-3.4
    environment:
      POSTGRES_DB: dhis2
      POSTGRES_USER: dhis
      POSTGRES_PASSWORD: dhis
    volumes:
      - db-data:/var/lib/postgresql/data
    restart: unless-stopped

  web:
    image: dhis2/core:2.40
    depends_on:
      - db
    ports:
      - "8080:8080"
    volumes:
      - ./dhis.conf:/opt/dhis2/dhis.conf:ro
      - dhis2-home:/opt/dhis2
    environment:
      JAVA_OPTS: "-Xms1g -Xmx3g"
    restart: unless-stopped

volumes:
  db-data:
  dhis2-home:
```

### 1b. dhis.conf

```
connection.dialect = org.hibernate.dialect.PostgreSQLDialect
connection.driver_class = org.postgresql.Driver
connection.url = jdbc:postgresql://db/dhis2
connection.username = dhis
connection.password = dhis
connection.pool.max_size = 20
```

### 1c. Start

```bash
docker compose up -d
```

Wait ~90 seconds for DHIS2 to initialize, then confirm it is running:

```bash
curl -s -o /dev/null -w "%{http_code}" \
  -u admin:district http://localhost:8080/api/system/info
# → 200
```

Default admin credentials: `admin` / `district`

## Phase 1d — Import EPI Metadata Package

**Package:** WHO/HISP EPI Aggregate Metadata Package v1.1.0 for DHIS2 2.40  
**File:** `IMM_AGG_COMPLETE_1.1.0_DHIS2.40_patched.json`

The file is a patched version of the official package. Three changes were made from the original before import:

1. Removed the embedded user object (`vUeLeQMSwhN`) from the `users` array — a fresh DHIS2 instance does not have this user, which caused sharing resolution failures.
2. Removed two `jobConfigurations` entries that referenced the same user.
3. Reset `sharing` on all 572 objects to `{"external": false, "public": "rwrw----", "users": {}, "userGroups": {}}` so the admin account has read-write access without group membership.

Import must be done in **three sequential steps** because DHIS2 resolves `DataSet.categoryCombo` references from the Hibernate session; importing everything in one request causes a null-reference error on `DataSet.periodType`.

```bash
# Step 1 — base metadata (categories, data elements, indicators, etc.)
curl -s -u admin:district \
  -X POST "http://localhost:8080/api/metadata?importMode=COMMIT&mergeMode=REPLACE&skipSharing=false" \
  -H "Content-Type: application/json" \
  -d @IMM_AGG_COMPLETE_1.1.0_DHIS2.40_patched.json \
  | python3 -c "import sys,json; r=json.load(sys.stdin); print(r.get('status'), r.get('stats',{}))"
```

After import, unlock the two EPI datasets for data entry (they import with restricted public access by default):

```bash
docker compose exec db psql -U dhis -d dhis2 -c \
  "UPDATE dataset SET publicaccess='rwrw----'
   WHERE uid IN ('vI4ihClxSm4','jqSaKxtj8IA');"
```

Woreda names are taken from OCHA HDX COD-AB Ethiopia (April 2026). Facility names follow standard Ethiopian health system naming conventions.

```bash
python3 build_ethiopia.py
```

Output: `ethiopia_uid_map.json` (all UIDs for levels 1–5).

After running, grant the admin user data entry and view scope over the new Ethiopia root:

```bash
ETH_UID=$(python3 -c "import json; print(json.load(open('ethiopia_uid_map.json'))['ethiopia']['uid'])")

docker compose exec db psql -U dhis -d dhis2 -c "
  INSERT INTO usermembership(userinfoid, organisationunitid)
  SELECT u.userinfoid, o.organisationunitid
  FROM userinfo u, organisationunit o
  WHERE u.username='admin' AND o.uid='${ETH_UID}'
  ON CONFLICT DO NOTHING;

  INSERT INTO userdatavieworgunits(userinfoid, organisationunitid)
  SELECT u.userinfoid, o.organisationunitid
  FROM userinfo u, organisationunit o
  WHERE u.username='admin' AND o.uid='${ETH_UID}'
  ON CONFLICT DO NOTHING;"

curl -s -u admin:district -X POST \
  "http://localhost:8080/api/maintenance?cacheClear=true"
```

### Walkthrough: seeded data quality issues (April 2026)

All three issues are in period **April 2026**. Switch to that period after navigating to each facility below.

**Issue 1 — Missing report**
- Facility: **Bichena Health Post** (Dabat Woreda → expand `+`)
- Expected: the entire form is blank — no data was submitted for April 2026
- This simulates a facility that failed to report for the month

**Issue 2 — Statistical outlier**
- Facility: **Limalimo Health Post** (Janamora Woreda → expand `+`)
- Expected: BCG `< 1 year` = **350**
- Normal range for a health post is 30–45. This value is ~10× the expected maximum, simulating a data entry error or accidental duplication.

**Issue 3 — DTP1/DTP3 inconsistency**
- Facility: **Adi Goshu Health Post** (Addi Arekay Woreda)
- Expected: DPT-HepB-HIB 1 `< 1 year` = **45**, DPT-HepB-HIB 3 `< 1 year` = **80**
- DTP3 (third dose) cannot logically exceed DTP1 (first dose) because dropout only goes one direction. This simulates a transposition or reporting error.

## Key technical notes

**Why DHIS2 2.40 and not 2.41:** DHIS2 2.41.x has a regression where `POST /api/dataSets` with `"periodType": "Monthly"` throws `org.hibernate.PropertyValueException: not-null property references a null or transient value : DataSet.periodType`. The EPI Aggregate Metadata Package v1.1.0 targets 2.40, so 2.40 is used throughout.

**Why three-step metadata import:** A single bulk import processes DataSet objects before CategoryCombos are fully loaded into the Hibernate session, causing a cascading null on `DataSet.periodType`. Splitting into base → datasets → rules resolves this.

**Why SQL fix for admin org unit scope:** DHIS2 does not automatically add newly created root org units to the admin user's data entry or view scope. The `usermembership` and `userdatavieworgunits` inserts are required even for the superuser account, followed by a cache flush.

**Why the dataset `publicaccess` SQL fix:** The EPI package imports datasets with `publicaccess = 'r-r-----'` (read-only). Setting it to `'rwrw----'` allows the admin account and role-based users to submit data.

## Resetting from scratch

To wipe everything and start over, remove all DHIS2 Docker volumes:

```bash
docker compose down
docker volume ls | grep -E 'db-data|dhis2-home' | awk '{print $2}' | xargs docker volume rm
docker compose up -d
```

Then re-run phases 1d through 5 in order.
